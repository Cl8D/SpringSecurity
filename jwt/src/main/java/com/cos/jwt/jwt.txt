## 세션 이용
클라이언트 -> 서버로 ID, PW를 최초로 보내면,
서버는 session 영역에 sessionId를 만들고 해당 id를 위한 영역을 만든다.
서버는 이때 세션ID를 리턴해준다.
이때, 클라이언트는 웹 브라우저를 요청했기 때문에 프로그램 쿠키 영역에 세션 id를 저장한다.

그리고 그 다음 요청부터 세션 ID를 함께 보내주며 인증을 처리한다.
-> 그러나, 클라이언트가 자바스크립트로 요청을 하게 되면
쿠키는 기본 정책이 동일한 도메인에서만 요청이 올 때 발동되기 때문에 (SOP)
도메인이 달라지면 쿠키를 아예 안 받아준다.
특히나 쿠키는 http only여서 자바스크립트에서는 아예 건들 수도 없게 되는 것.

그래서 서버에서 쿠키 http only를 false로 해주면 받을 수는 있지만,
이러면 보안상의 문제가 발생한다.
그리고, 쿠키가 많아질수록 관리하기도 힘들어진다.

--

그래서, 헤더에 authorization이라는 키 값에 인증 정보를 넣는 방식 중에,
거기에 id, pw를 넣어서 요청하는 게 바로 http basic 방식.
단, 이때 암호화가 안 되어서 중간에 노출이 될 수 있고,
이를 방지하기 위해 https를 쓰면 된다.

--

우리는 이것 대신에, authorization에 토큰을 넣는 방식을 사용할 것이다.
이때 토큰은 노출이 되면 안 되지만 id, pw 정보 자체가 나타내지는 않으니까
노출이 되어도 그나마 안전하다.
이게 바로 bearer 방식.
